# API YaMDb

YaMDb — REST API для сбора отзывов и оценок произведений. Пользователи могут
просматривать произведения, публиковать по одному отзыву на каждое
произведение, ставить оценки от 1 до 10 и комментировать отзывы. Модераторы
управляют пользовательским контентом, а администраторы — пользователями,
произведениями, категориями и жанрами.

## Технологии

- Python 3.10+
- Django 5.1
- Django REST Framework 3.15
- Simple JWT
- SQLite
- pytest и pytest-django
- Postman / Newman

## Установка

Клонируйте репозиторий и перейдите в его директорию:

```bash
git clone <repository-url>
cd api-yamdb
```

Создайте и активируйте виртуальное окружение.

Linux и macOS:

```bash
python3 -m venv venv
source venv/bin/activate
```

Windows PowerShell:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

Установите зависимости и выполните миграции:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
python api_yamdb/manage.py migrate
```

Запустите сервер разработки:

```bash
python api_yamdb/manage.py runserver
```

API будет доступен по адресу `http://127.0.0.1:8000/api/v1/`.

## Аутентификация

В YaMDb используется JWT-аутентификация. Регистрация выполняется в два этапа.

### 1. Получение кода подтверждения

```http
POST /api/v1/auth/signup/
Content-Type: application/json

{
  "email": "user@example.com",
  "username": "user"
}
```

Ответ содержит `email` и `username`. Письмо с кодом подтверждения сохраняется
в директории `api_yamdb/sent_emails/`. При повторном запросе с теми же
`username` и `email` создаётся новый код, а предыдущий становится
недействительным.

### 2. Получение JWT-токена

```http
POST /api/v1/auth/token/
Content-Type: application/json

{
  "username": "user",
  "confirmation_code": "код-из-письма"
}
```

Успешный ответ:

```json
{
  "token": "eyJ..."
}
```

Передавайте токен в заголовке защищённых запросов:

```http
Authorization: Bearer <token>
```

## Пользовательские роли

| Роль | Права |
| --- | --- |
| Аноним | Просмотр произведений, категорий, жанров, отзывов и комментариев |
| Пользователь | Права анонима, создание отзывов и комментариев, управление своим контентом |
| Модератор | Права пользователя, редактирование и удаление любых отзывов и комментариев |
| Администратор | Полное управление контентом и пользователями |
| Суперпользователь | Те же права API, что и у администратора |

## Эндпоинты API

Все адреса используют префикс `/api/v1/`.

### Аутентификация и пользователи

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| POST | `/auth/signup/` | Все | Регистрация или получение нового кода подтверждения |
| POST | `/auth/token/` | Все | Обмен имени пользователя и кода подтверждения на JWT |
| GET | `/users/` | Администратор | Список пользователей; поиск `?search=username` |
| POST | `/users/` | Администратор | Создание пользователя |
| GET | `/users/{username}/` | Администратор | Получение пользователя |
| PATCH | `/users/{username}/` | Администратор | Частичное обновление пользователя |
| DELETE | `/users/{username}/` | Администратор | Удаление пользователя |
| GET | `/users/me/` | Авторизованный | Получение собственного профиля |
| PATCH | `/users/me/` | Авторизованный | Обновление профиля; поле `role` доступно только для чтения |

### Категории

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| GET | `/categories/` | Все | Список категорий; поиск `?search=name` |
| POST | `/categories/` | Администратор | Создание категории |
| DELETE | `/categories/{slug}/` | Администратор | Удаление категории |

Изменение категорий методами `PUT` и `PATCH` не поддерживается.

### Жанры

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| GET | `/genres/` | Все | Список жанров; поиск `?search=name` |
| POST | `/genres/` | Администратор | Создание жанра |
| DELETE | `/genres/{slug}/` | Администратор | Удаление жанра |

Изменение жанров методами `PUT` и `PATCH` не поддерживается.

### Произведения

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| GET | `/titles/` | Все | Список произведений |
| POST | `/titles/` | Администратор | Создание произведения |
| GET | `/titles/{id}/` | Все | Получение произведения |
| PATCH | `/titles/{id}/` | Администратор | Частичное обновление произведения |
| DELETE | `/titles/{id}/` | Администратор | Удаление произведения |

Фильтры списка произведений:

- `?name=<название>`
- `?year=<год выпуска>`
- `?category=<slug категории>`
- `?genre=<slug жанра>`

В запросах `category` передаётся как slug, а `genre` — как список slug. В
ответах категория и жанры представлены вложенными объектами. Поле `rating`
доступно только для чтения и содержит округлённую среднюю оценку отзывов.
Метод `PUT` не поддерживается.

Пример создания произведения:

```json
{
  "name": "Пример произведения",
  "year": 2024,
  "description": "Описание произведения",
  "category": "movie",
  "genre": ["drama"]
}
```

### Отзывы

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| GET | `/titles/{title_id}/reviews/` | Все | Список отзывов на произведение |
| POST | `/titles/{title_id}/reviews/` | Авторизованный | Создание отзыва |
| GET | `/titles/{title_id}/reviews/{review_id}/` | Все | Получение отзыва |
| PATCH | `/titles/{title_id}/reviews/{review_id}/` | Автор, модератор, администратор | Изменение отзыва |
| DELETE | `/titles/{title_id}/reviews/{review_id}/` | Автор, модератор, администратор | Удаление отзыва |

Пользователь может оставить только один отзыв на каждое произведение. Поле
`score` должно содержать целое число от 1 до 10. Метод `PUT` не поддерживается.

### Комментарии

| Метод | Эндпоинт | Доступ | Описание |
| --- | --- | --- | --- |
| GET | `/titles/{title_id}/reviews/{review_id}/comments/` | Все | Список комментариев к отзыву |
| POST | `/titles/{title_id}/reviews/{review_id}/comments/` | Авторизованный | Создание комментария |
| GET | `/titles/{title_id}/reviews/{review_id}/comments/{comment_id}/` | Все | Получение комментария |
| PATCH | `/titles/{title_id}/reviews/{review_id}/comments/{comment_id}/` | Автор, модератор, администратор | Изменение комментария |
| DELETE | `/titles/{title_id}/reviews/{review_id}/comments/{comment_id}/` | Автор, модератор, администратор | Удаление комментария |

Метод `PUT` не поддерживается.

## Пагинация

Эндпоинты списков возвращают по 10 объектов на странице:

```json
{
  "count": 1,
  "next": null,
  "previous": null,
  "results": []
}
```

Для перехода между страницами используйте query-параметр `page`.

## Тестирование

Запуск всех pytest-тестов из корня репозитория:

```bash
python -m pytest
```

Postman-коллекция запускается по инструкции из
[`postman_collection/README.md`](postman_collection/README.md). Для
аутентификации используется двухэтапный сценарий:

1. Запустите папку `registration // No Auth/get_confirmatior_codes`.
2. Откройте четыре письма в `api_yamdb/sent_emails/`.
3. Запишите коды в `Current value` переменных `*ConfirmationCode` коллекции.
4. Запустите коллекцию, отключив первые четыре запроса `signup`.

Не запускайте запросы `signup` повторно после сохранения кодов: каждый успешный
запрос создаёт новый код и делает предыдущее значение недействительным.
